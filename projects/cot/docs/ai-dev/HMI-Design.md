## 1. 设计概述

### 1.1 设计理念
- **简洁优先**: 11寸触摸屏，信息密度适中
- **状态感知**: 一眼看清系统当前状态和潜在风险
- **操作安全**: 关键操作需要二次确认，防止误触
- **层级清晰**: 高频功能在首页，低频功能深入菜单

### 1.2 硬件约束
| 参数 | 规格 | 设计影响 |
|------|------|----------|
| 屏幕尺寸 | 11寸 | 单屏信息密度可控制在8-12个核心元素 |
| 分辨率 | 1920×1080 | 字体最小16px，按钮最小48×48px |
| 触摸方式 | 电容触控 | 支持滑动，但主要操作为点击 |
| 使用环境 | 户外机柜 | 高对比度配色，支持亮度调节 |

### 1.3 色彩系统
```
主色调: #1890FF (科技蓝) - 主操作、当前状态
辅助色: #52C41A (绿色)   - 正常、运行中
警告色: #FAAD14 (橙色)   - 告警、注意
危险色: #F5222D (红色)   - 故障、停止
中性色: #8C8C8C (灰色)   - 离线、未激活
背景色: #141414 (深灰)   - 深色主题，适合工业环境
卡片色: #1F1F1F (近黑)   - 内容区域
```

## 4. 交互设计规范

### 4.1 手势操作
| 手势 | 功能 | 适用场景 |
|------|------|----------|
| 点击 | 选中/确认 | 按钮、列表项 |
| 长按 | 快捷操作 | 设备卡片呼出更多选项 |
| 滑动 | 页面/列表滚动 | 长列表、趋势图 |
| 下拉 | 刷新数据 | 设备列表、告警列表 |
| 捏合 | 缩放 | 功率流向图放大 |

### 4.2 二次确认场景
以下操作需要弹窗确认：
- 模式切换（绿电↔手动↔备电）
- 柴发启停
- PCS批量停机
- 参数修改（影响系统运行的关键参数）
- 系统关机/重启

### 4.3 错误提示
```
┌────────────────────────────────────────┐
│ ❌ 操作失败                            │

│                                       │
│  原因: 当前存在紧急告警未确认           │
│  (绝缘故障告警)                        │
│                                       │
│  [查看告警]           [我知道了]        │
└────────────────────────────────────────┘
```
