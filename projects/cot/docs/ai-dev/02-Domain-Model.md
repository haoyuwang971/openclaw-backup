- **新增**: 功率平衡方程 P_bat = P_pv_actual - P_clu/η

**历史更新** (v1.0 → v1.1):
- 根据完整设备协议更新所有寄存器地址和倍率
- MPPT控制逻辑更新: 主机分配 → EMS直接控制
- BAM协议更新: 占位符 → 高特ESMU V2.1 Modbus TCP
- 液冷机补充: 英维克A8完整协议
- 电表区分: 东鸿(储能/负载/柴发) vs 安科瑞(关口表)

## 2. 系统拓扑

### 2.1 电气拓扑结构（⚠️ 已纠正）

```
┌─────────────────────────────────────────────────────────────────┐
│                         交流母线 (AC Bus)                        │
│  ┌─────────┐    ┌─────────┐    ┌─────────┐                     │

│  │1#2#3#   │    │4#5#     │    │         │                     │
│  │(375kW)  │    │(250kW)  │    │         │                     │
│  └────┬────┘    └────┬────┘    └─────────┘                     │
│       │              │                                         │
│       └──────────────┴──────────────┐                          │
│                                     ▼                          │
│                              ┌──────────┐                      │

│                              └────┬─────┘                      │

│                              ┌────┴─────┐                      │

│                     ┌─────────────┴─────────────┐              │
│                     ▼                             ▼              │
│                ┌─────────┐                 ┌─────────┐         │
│                │  市电   │                 │  柴发   │         │
│                └─────────┘                 └─────────┘         │
└─────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────┐
│              簇1直流侧功率流 (DC Side - Cluster 1)               │
│                                                                  │
│    MPPT 1~4 (400kW) ───┐                                        │
│        直流输出         │──→ 直流母线 ──→ PCS 1~3 (AC/DC双向)   │
│    电池 1~2 (260kWh) ──┘           ↓ 逆变/整流                  │
│    (BAM管理)                      交流侧                        │
│                                                                  │
│  ⚠️ 关键纠正：                                                    │
│  - MPPT直接输出直流到母线，不是通过PCS！                         │
│  - 光伏功率在直流侧与电池直接交换                                │
│  - PCS只负责AC/DC变换（交流侧与直流侧之间）                      │
│  - 直流侧功率平衡：P_bat = P_pv - P_clu/η                        │
└─────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────┐
│              簇2直流侧功率流 (DC Side - Cluster 2)               │
│                                                                  │
│    MPPT 5~7 (300kW) ───┐                                        │
│        直流输出         │──→ 直流母线 ──→ PCS 4~5 (AC/DC双向)   │
│    电池 3~4 (260kWh) ──┘                                        │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

**拓扑特征**:
- **两簇PCS交流侧并联**，共接交流母线，与负载、STS连接
- **每簇独立直流母线**：MPPT和电池**并联**在直流母线上
- **MPPT直接输出直流电到直流母线**，不是通过PCS传递给电池
- **PCS是AC/DC双向变换器**，连接交流侧和直流侧
- **直流侧功率平衡**：电池功率 = 光伏实际功率 - PCS功率/效率
- **ATS**：市电优先级高，保证只有一个电压源输入（市电/柴发二选一）
- **STS**：控制并网/离网切换

**⚠️ 常见误解纠正**:
| 错误理解 | 正确理解 |
|----------|----------|
| 光伏→PCS→电池（交流路径） | 光伏→直流母线→电池（直流路径） |
| PCS把光伏电充给电池 | PCS只负责AC/DC变换，光伏直流直接到电池 |
| MPPT功率经过PCS转换 | MPPT直流输出直接到直流母线 |

### 2.5 状态空间模型定义（⚠️ 新增 - 基于MPC学术分析）

为支持MPC控制器设计，建立以下状态空间模型：

#### 状态变量 (x)
```
x = [SOC₁, SOC₂]ᵀ ∈ R²
```
| 变量 | 物理意义 | 单位 | 范围 |
|------|----------|------|------|
| SOC₁ | 簇1电池荷电状态 | - | [0.10, 0.90] |
| SOC₂ | 簇2电池荷电状态 | - | [0.10, 0.90] |

#### 控制输入 (u)
```
u = [P_clu₁, P_clu₂, P_pv_limit₁, P_pv_limit₂]ᵀ ∈ R⁴
```
| 变量 | 物理意义 | 单位 | 范围 | 说明 |
|------|----------|------|------|------|
| P_clu₁ | 簇1 PCS功率 | kW | [-375, +375] | 正放电，负充电 |
| P_clu₂ | 簇2 PCS功率 | kW | [-250, +250] | 正放电，负充电 |
| P_pv_limit₁ | 簇1光伏限功 | kW | [0, 400] | 限制MPPT输出 |
| P_pv_limit₂ | 簇2光伏限功 | kW | [0, 300] | 限制MPPT输出 |

#### 扰动输入 (d)
```
d = [P_pv₁, P_pv₂, P_load]ᵀ ∈ R³
```
| 变量 | 物理意义 | 单位 | 说明 |
|------|----------|------|------|
| P_pv₁ | 簇1光伏可用功率 | kW | 来自MPPT测量 |
| P_pv₂ | 簇2光伏可用功率 | kW | 来自MPPT测量 |
| P_load | 负载功率 | kW | 来自关口表测量 |

#### 状态方程（⚠️ 关键纠正）

**直流侧功率平衡**（单簇）：
```
P_bat = P_pv_actual - P_clu/η
```
其中：
- P_bat: 电池功率（**正充电，负放电**），kW
- P_clu: PCS功率（**正放电，负充电**），kW
- η: PCS效率（约98%）
- P_pv_actual: 实际光伏功率 = min(P_pv, P_pv_limit)，kW

**⚠️ 重要说明**：
- **光伏直接输出直流电到直流母线**，不是通过PCS
- PCS只负责AC/DC双向变换，连接交流侧和直流侧
- 光伏和电池在直流侧是**并联关系**，功率直接交换

**SOC动态方程**（连续时间）：
```
dSOC/dt = -P_bat / (V_nom × Q_nom × 3.6)  [%/h]
```
其中：
- V_nom: 电池额定电压 = 870V
- Q_nom: 电池额定容量 = 261Ah

**离散化状态方程**（前向欧拉法，Δt=5s）：
```
SOC₁(k+1) = SOC₁(k) - [P_clu₁(k)/η - P_pv_actual₁(k)] × Δt / (V×Q×3.6)
SOC₂(k+1) = SOC₂(k) - [P_clu₂(k)/η - P_pv_actual₂(k)] × Δt / (V×Q×3.6)
```

#### 输出方程

**交流侧功率平衡**：
```
P_grid = P_load - P_clu₁ - P_clu₂
```

**完整输出向量**：
```
y = [P_grid, SOC₁, SOC₂]ᵀ
```

#### 约束条件

**不等式约束**：
```
SOC₁ ≥ 0.70          (备电约束)
SOC₂ ≥ 0.70
SOC₁ ≤ 0.90          (SOC上限)
SOC₂ ≤ 0.90
0 ≤ P_grid ≤ 500     (需量约束 + 禁止逆流)
```

**边界约束**：
```
-375 ≤ P_clu₁ ≤ +375
-250 ≤ P_clu₂ ≤ +250
0 ≤ P_pv_limit₁ ≤ P_pv₁ × 1.2
0 ≤ P_pv_limit₂ ≤ P_pv₂ × 1.2
```

- 其余故障: 关机状态下发**故障复位指令(0x1400 Bit15=1)**

#### 3.1.6 PCS控制流程

**离网模式开机序列**:
1. 设置 PCS 1# 功率 = 0，等待建立离网电压
2. 依次启动 PCS 2# → 3# → 4# → 5#

**关机序列**:
1. 设置 PCS 2#~5# 功率 = 0，依次停机
2. 最后停止 PCS 1# (**VF主机必须最后退出**)

**功率分配**:
- 簇1 (PCS 1#~3#): 主机PCS 1#控制总功率，均分给3台
- 簇2 (PCS 4#~5#): 主机PCS 4#控制总功率，均分给2台

#### 3.2.4 MPPT控制逻辑 (v1.1更新)

⚠️ **重要变更**: EMS直接对每台MPPT独立设置功率百分比，不再通过主从分配。

**控制架构**:
| 簇 | MPPT设备 | 功率控制方式 | 功率边界 |
|----|----------|--------------|----------|
| **簇1** | 1#、2#、3#、4# | EMS单独控制每台MPPT的46110功率百分比 | [0kW, 400kW] |
| **簇2** | 5#、6#、7# | EMS单独控制每台MPPT的46110功率百分比 | [0kW, 300kW] |

**功率分配计算**:
1. 计算簇总PV功率限制: P_limit_total = 95% × (簇电池充电需求 + 簇PCS实时功率)
2. EMS根据各MPPT状态（在线/故障/功率余量）将P_limit_total分配给各台
3. 各台独立设置46110功率百分比: **百分比 = (分配功率 / 基准功率) × 100%**

**重要约束**:
- MPPT **只能单向控制，不能放电**（功率≥0）
- 限功率通过**百分比实现**，而非直接功率设定值
- **EMS需维护7台MPPT各自的功率限制状态**
- 固化参数时，60/120kW机型**必须关机**，320kW无需关机

**EMS控制流程**:
1. 读取 45200 状态字，确认bit8=1（允许启机）
2. 写 46001=1 开机
3. 根据功率分配策略，**分别写各台46110功率百分比**
4. 读取各台45217输入侧总功率，确认实际输出功率
5. 停机时写 46001=0 关机

#### 3.2.5 MPPT运行模式与约束（⚠️ 重要）

**自动运行模式** (出厂默认开启):
- 输入输出电压正常且无故障时，MPPT**自动开机**
- 重新上电后**自动恢复**运行
- 故障复位后**自动重新**开机

⚠️ **关键约束 - EMS必须严格遵守**:
| 场景 | 行为 | EMS应对策略 |
|------|------|------------|
| **通讯关机后** | 必须用**通讯开机**才能恢复；未通讯开机不会自动启动 | EMS关机后需记录状态，下次开机必须先发46001=1 |
| **执行通讯开机后** | 恢复自动运行模式 | 执行一次46001=1后，后续可依赖自动模式 |
| **自动模式关闭** | 每次开关机都必须通过通讯命令操作 | 需通过寄存器配置检查当前模式 |

> **设计要点**: EMS的MPPT控制逻辑必须区分"首次开机"和"恢复运行"。首次上电或EMS主动关机后，必须显式发送46001=1；故障恢复后的自动重启则无需干预。

**开机条件** (必须同时满足):
1. 辅源供电正常 (**由光伏输入侧供电**，非电池侧)
2. 输入侧光伏电压在可工作范围内
3. 输出侧电池电压在可工作范围内

**拓扑约束** (Boost电路):
- 正常工作: 输入光伏电压 < 输出电池电压 (需留余量)
- 光伏板选型建议: 开口电压(含低温系数) ≤ **3.45V × 总电池Cell数**
- 电压过低时: 可利用二极管直通特性直接充电 (**此时MPPT不可控**)

**组串配置** (120kW机型):
- 4个独立单元模块，每模块接**3路组串**
- 3路组串在单元模块输入处并联
- **同一单元内3路组串配置必须一致**，否则形成环流

**常见故障排查速查表**:
| 故障/告警 | 可能原因 | 排查重点 | EMS处理建议 |
|-----------|---------|---------|------------|
| "开机条件不满足" | 输出侧电池电压过低/过高 | 检查电池簇输出开关是否闭合 | 监控电池电压，低电压时禁止开机指令 |
| "AD零漂过大" | 同一单元内组串配置不一致 | 统一组串配置；或断电2min重试 | 告警提示运维检查 |
| "正/负极对地绝缘阻抗低" | 电缆绝缘异常 | 开机前查输入/输出接线；运行后查输出电缆破损 | 触发绝缘故障告警 |
| "辅源供电异常" | 光伏输入侧无电 | 检查光伏输入 | 夜间或阴天时MPPT无法工作，属正常现象 |

**实验室测试模式**:
- 接多个单元模块时，必须改为**共源模式**，否则多模块寻优会拉垮模拟源
- 共源模式不保存，重新上电即失效

```

**推荐设定值**:
- 制冷点: 25.0℃ (寄存器值=250)
- 加热点: 15.0℃ (寄存器值=150)

- 维护: 0x0401 = 1 (自循环)

- 0xE715 ≥ 1: EMS触发告警通知
- 0xE715 ≥ 2: EMS评估是否需要降功率或停机
- 0xE715 = 3: EMS建议紧急停机

**重要约束**:
- 液冷机与DC柜一一对应（1#~4#）
- 温度设定需根据BAM电池温度反馈动态调整
- 心跳(0xE71C)需定期读取监控通信状态

### 4.1 数据采集周期

```
┌─────────────────────────────────────────────────────────────┐
│                      数据采集周期                            │

```

```
EMS Controller
