## 三、关键技术决策 (已固化)

### 3.1 系统架构
- **硬件**: ARM 4核/2GB RAM/8GB EMMC (WL-EMS-1000-M)
- **拓扑**: 交流耦合，两簇直流母线独立 + 交流母线并联
- **设备**: 5×PCS + 4×DC柜 + 7×MPPT + 柴发 + STS

### 3.2 簇(Cluster)定义
| 簇 | PCS | MPPT | DC柜 | BAM通信 |
|----|-----|------|------|---------|
| **簇1** | 1#~3# (375kW) | 1#~4# (400kW) | 1#~2# | TCP1 |
| **簇2** | 4#~5# (250kW) | 5#~7# (300kW) | 3#~4# | TCP2 |

### 3.3 四层SOC阈值 (核心控制逻辑)

| 阈值 | 数值 | 滞环 | 功能场景 |
|------|------|------|----------|
| **柴发启动** | 20% | - | 保命线，强制启动柴发 |
| **备电恢复** | 70% | ±1% (69%/71%) | 市电快充恢复备电能力 |
| **柴发停机** | 80% | - | 可停机条件 |
| **绿电消纳** | 90% | ±2% (88%/92%) | PCS放电做负载跟随 |

**充电策略**:
- SOC < 69%: 市电快充（仅需量约束）
- 71%~90%: 只用光伏充电（PCS=0）
- ≥ 92%: 绿电消纳模式（防逆流生效）

### 3.4 功率方向定义 (已统一)

| 变量 | 正方向 | 负方向 |
|------|--------|--------|
| **P_bat** | 充电（流入电池） | 放电（流出电池） |
| **P_pcs** | 放电（逆变DC→AC） | 充电（整流AC→DC） |
| **P_pv** | 发电（总是正值） | - |

**功率平衡方程**: `P_bat = P_pv - P_pcs/η`

### 3.5 状态机核心

**5个系统状态**: 待机(STANDBY) → 离网(OFFGRID) → 并市电(GRID) → 并柴发(DIESEL) → 故障(FAULT)

**6个切换条件 (C1-C6)**:
- C1: 系统就绪（无故障）
- C2: 故障触发
- C3: 市电就绪（压差<5%，频差<0.5Hz，相角<10°）
- C4: 柴发就绪（电压/频率/相位满足）
- C5: 离网条件（市电异常或STS分闸）
- C6: 待机条件（PCS全停机）

**扫描周期**: 100ms，防抖 300ms (3周期)

### 3.6 通信架构

**南向接入**:
- PCS ×5: RS485-4, 3s轮询, 地址0x11-0x15
- MPPT ×7: RS485-7, 3s轮询, 地址0x01-0x07  
- BAM ×2: TCP1/TCP2, 1s轮询
- STS: RS485-3, 地址0x21
- 电表: 关口表(Modbus-7) + 柴发表(Modbus-8)
- 柴发: DO-1启停控制, DI反馈

**北向接入**:
- 4G/5G MQTT → 储能云平台
- WebSocket → HMI实时推送

**东西向**:
- 并机以太网同步

## 五、经验教训

### 5.1 功率方向反复修正
- **问题**: 功率平衡方程多次写错
- **根因**: 缺乏物理拓扑锚点，凭直觉写公式
- **解决**: 建立统一方向定义，用场景验证

### 5.2 专业助手能力建设
- **问题**: 知识结构不牢固，修改时不会主动溯源
- **改进**: 修改前强制查01-PRD、查拓扑、一致性检查

### 5.3 文档命名混乱
- **问题**: 05/06文档覆盖冲突
- **解决**: 明确职责分离 → 05功能规格、06技术设计
