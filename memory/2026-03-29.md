- **Key Constraint**: Strict anti-backflow, demand control, no power export to grid

### EMS Strategy Key Points

#### 1. Mode Switching State Machine
- Grid-connected → Off-grid battery (fast STS switch, <20ms typical)
- Off-grid battery → Grid-connected (ATS + STS synchronization)
- Off-grid battery → Off-grid with diesel (EMU starts diesel, ATS + STS sync)
- Off-grid with diesel → Off-grid battery (stop diesel, STS switches)
- Off-grid with diesel → Grid-connected (ATS prioritizes grid)

#### 2. PCS Operating Modes
- **Grid-connected**: Current source, power control
- **Off-grid**: Voltage source, voltage/frequency establishment
- **With diesel**: Current source, diesel as voltage source

#### 3. Cluster SOC Balancing (Grid charging)
Formula: P_cluster = P_limit × (ΔSOC_cluster / ΣΔSOC)
- Balance based on distance to charge DOD limit
- Example: Cluster 1 needs 30%, Cluster 2 needs 60% → split 33.3% : 66.7%

#### 4. PV Power Limiting
P_limit = P_battery_demand + P_PCS_realtime - safety_margin
- Must follow PCS power changes in real-time
- Prevents overcharging and backflow

#### 5. SOC Threshold Control (Grid-connected)
| Range | PCS Strategy | Load Source | Battery Source |
|-------|-------------|-------------|----------------|
| DOD < SOC < Threshold1 | Load following | PV | PV |
| Threshold1 < SOC < Threshold2 | No charge/discharge | Grid | PV |
| Threshold2 < SOC < DOD | Grid charging | Grid | Grid |

Example thresholds:
- Night/overcast: 94% / 90%
- Morning/evening: 80% / 70%
- Noon: 70% / 60%

#### 6. Diesel Start/Stop Logic
- **Start**: SOCmin < start_SOC AND no grid AND insufficient PV
- **Stop**: Grid restored OR SOC ≥ stop_SOC
- **Constraint**: No charging battery from diesel (PCS power = 0)

#### 7. Hysteresis Control
All SOC judgments include hysteresis to prevent frequent switching:
- "> setpoint" means "> setpoint + hysteresis"
- "< setpoint" means "< setpoint - hysteresis"

## EMS Control Loop Architecture Discussion

### Three Key Relationships Explained
User asked about the relationship between:
1. **PCS command cycle period**
2. **PV power limiting response speed**
3. **SOC balancing smoothness**

**Key Insight**: These three form a control loop hierarchy:
- **PV limiting needs FAST response** (100-200ms) → PV changes rapidly (clouds)
- **SOC balancing needs SLOW response** (1-5s) → Avoid power oscillation
- **PCS command needs MEDIUM rate** (200-500ms) → With ramp rate limiting

**Recommended ROS2 Node Design**:
```
EMS主节点 (200ms周期)
    ├─► 光伏限功节点 (100ms，实时话题 /pv_limit)
    ├─► SOC均衡节点 (2s周期)
    └─► 功率合成 (200ms，加斜坡限制)
```

**No load priority shedding** for this project.
