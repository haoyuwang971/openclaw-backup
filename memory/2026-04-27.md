## 结构化需求分析方法与 Cot DFD 的对照解释

用户询问书中 2.3 章节（结构化需求分析方法）如何结合 Cot 项目理解。

我给出的核心解释：

结构化需求分析三大工具在 Cot DFD 中的对应：

| 工具 | 红宝书定义 | Cot 文档对应 |
|------|-----------|-------------|
| DFD 数据流图 | 数据从哪里来、经过什么加工、到哪里去 | 第2~4章（Context/Level1/Level2） |
| 数据字典 | 对数据流、存储、加工的精确定义 | 第5章（数据字典） |
| 加工小说明 | 每个加工内部的处理逻辑 | 各Level2子进程的功能说明表 |

分层对应关系：
- Level 0 → 上下文图（系统边界，10个外部实体）
- Level 1 → 按功能域拆8个子系统（P1~P8）
- Level 2 → 复杂加工继续分解（如P3拆到P3.7，P4拆到P4.7）

数据字典示例：
- `F4.1 防逆流上限` = max(0, 负载功率 - 5kW)，来源P4.1，去向P3.6，频率200ms
- `D1 实时数据库` = 所有设备最新测点值，内存读写，256MB，持续更新

加工小说明示例（结构化语言）：
```
IF (两簇最小SOC ≥ 92%) AND (并网状态 = true) THEN
    目标PCS功率 = 负载功率 × 95%
ELSE IF (两簇最小SOC < 88%) THEN
    目标PCS功率 = 0
ELSE
    维持当前PCS功率  // 滞环区
ENDIF
```

三者关系：DFD 告诉"数据怎么流"，数据字典告诉"数据是什么"，加工小说明告诉"加工怎么做"，三者构成完整的系统逻辑模型。

Cot 文档对红宝书方法的工程化扩展：
- 外部实体包含设备（PCS/MPPT等）而不仅是人和系统
- 按实时性分级（Hard/Firm/Soft/Best Effort）
- 数据字典增加频率、周期、来源去向，便于通信协议设计
- 增加时序图体现时间维度
